--- SECTION 16: Polymorphism with toString() method ---
Q: Observe what happens when iterating through an ArrayList of Media and calling toString(). Explain in detail.
A: 
- Observation: When we loop through the List<Media> and call `m.toString()`, the program correctly prints the specific format of a Book, a CD, or a DVD, depending on what the object actually is.
- Explanation: This is an illustration of Runtime Polymorphism (Dynamic Method Dispatch) in Java. Although the reference variable `m` is of type `Media`, Java looks at the ACTUAL object type stored in memory at runtime (e.g., a `Book` object). Since `Book`, `CompactDisc`, and `DigitalVideoDisc` all override the `toString()` method from the parent class `Media`, Java will dynamically bind and call the overridden `toString()` method of the specific child class.

--- SECTION 16: Reading Assignment ---
1. What are the advantages of Polymorphism?
A: 
- Code Reusability and Cleanliness: We can write a single piece of code (like a loop iterating over Media objects) that works with many different specific types of objects.
- Extensibility: We can easily add new types of Media (like VideoGame or Magazine) in the future. As long as they extend Media and override the necessary methods, the existing code (like the Cart's print method) will still work without any modifications.
- Flexibility: It allows a single interface to represent different underlying forms (data types).

2. How is Inheritance useful to achieve Polymorphism in Java?
A: Inheritance provides the foundational "IS-A" relationship required for polymorphism. By having subclasses (Book, CD, DVD) inherit from a common superclass (Media), we can use a superclass reference variable to point to any subclass object. Without this inheritance hierarchy, we could not treat different objects uniformly, and dynamic method overriding would not be possible.

3. What are the differences between Polymorphism and Inheritance in Java?
A: 
- Inheritance is about defining a structure and reusing code. It establishes a parent-child relationship (creating new classes from existing ones).
- Polymorphism is about behavior and flexibility. It is the ability of an object to take on many forms and allows us to perform a single action in different ways (calling the same method on different objects and getting specific behaviors). Inheritance provides the structure, Polymorphism utilizes that structure.

--- SECTION 17: Sort media in the cart ---
Suppose we use the Comparable interface approach instead of Comparator.
1. What class should implement the Comparable interface?
A: The parent class `Media` should implement the `Comparable<Media>` interface.

2. In those classes, how should you implement the compareTo() method to reflect the ordering that we want?
A: We would override `public int compareTo(Media other)` in the `Media` class. Inside, we would write the logic to compare the current object (`this`) with the `other` object (e.g., `return this.title.compareTo(other.getTitle());`).

3. Can we have two ordering rules of the item (by title then cost and by cost then title) if we use this Comparable interface approach?
A: No, we cannot. The Comparable interface is used to define the "natural ordering" of an object, which means a class can only have ONE `compareTo()` method implementation. If we need multiple distinct ways to sort the same objects, we MUST use the `Comparator` interface approach (which we implemented in this lab).

4. Suppose the DVDs have a different ordering rule from the other media types (title -> decreasing length -> cost). How would you modify your code to allow this?
A: We would create a specific Comparator class for DVDs, for example, `public class DVDComparator implements Comparator<DigitalVideoDisc>`. Inside the `compare(DigitalVideoDisc d1, DigitalVideoDisc d2)` method, we would implement the 3-level sorting logic: first compare titles, if equal, compare lengths in reverse order, if still equal, compare costs. Then we pass this specific comparator to `Collections.sort()` when dealing exclusively with a list of DVDs.
